# 德州扑克游戏开发需求

## 一、技术栈
- 语言：Python 3.10+
- 图形库：pygame
- 无外部网络依赖，纯本地单机运行

## 二、游戏模式
- **2人模式（Heads-Up / 精英战）**：1V1 对决，AI 使用比 MCTS 更聪明的算法
  - 可基于 CFR 思想的简化实现（手牌强度抽象 + 动作抽象 + 预训练策略表），或其他更优算法
  - 核心目标：2人模式下 AI 决策质量明显高于多人模式的 MCTS AI
  - 支持 人类 vs AI，也支持 AI vs AI（观战模式）
- **3-8人模式（多人桌）**：AI 使用蒙特卡洛模拟（MCTS）+ 性格矩阵
  - 每个 AI 玩家从预设角色池中随机选取，具有独立姓名和性格，模拟真人玩家
  - MCTS 每次决策模拟 500-2000 次，思考时间上限 3 秒（取先到者）

## 三、游戏规则
- **变体选择**：游戏开始时可选择
  - **No-Limit（无限注）**：加注无上限
  - **Pot-Limit（底池限注）**：加注不超过当前底池
  - **Fixed-Limit（限注）**：固定加注额度
- **短牌德州（Short Deck）**：当人数 < 5 人时可选
  - 牌组去掉 2-5（仅保留 6-A，共 36 张）
  - 同花 > 葫芦（牌型大小顺序调整）
  - A 可组成顺子 A-6-7-8-9
  - 其余规则同标准德州
- **盲注选择**：游戏开始时可选本局盲注数值
  - 预设档位：5/10、10/20、25/50、50/100
  - 支持自定义输入盲注金额
  - 可选是否有 ante
- **起始筹码**：玩家从银行取款带入游戏，取款金额需 ≥ 1 倍大盲
- **模式**：现金桌（Cash Game），盲注固定不递增
- **淘汰**：筹码归零即淘汰，游戏继续直到只剩一人
- **标准阶段**：Pre-flop → Flop → Turn → River → Showdown
- **可选动作**：Fold / Check / Call / Bet / Raise / All-in
- **最小加注**：不低于上一次加注幅度

## 四、AI 角色池与性格矩阵（多人模式）

### 角色池
- 预设 **30-50 个** AI 角色，每个角色有固定姓名、头像、性格参数
- 每个 AI 角色拥有独立的银行筹码（持久化存储）
- 每局开始时，从角色池中随机选取若干 AI 角色入座
- AI 角色带入游戏的筹码从其银行余额中扣除，游戏结束后结算回银行
- AI 角色筹码归零后该角色本局淘汰，银行余额为 0 的角色不参与选座

### 性格矩阵
每个 AI 角色具有以下性格维度（0.0-1.0 连续值，创建时固定）：
- **松紧度（Tight-Loose）**：影响入局频率，越松越愿意用边缘手牌入局
- **激进性（Passive-Aggressive）**：影响加注/诈唬频率，越激进越倾向于主动加注
- **诈唬倾向（Bluff Frequency）**：影响无强牌时的诈唬概率
- **跟注倾向（Call Tendency）**：影响面对加注时的弃牌/跟注/再加注决策
- **适应性（Adaptivity）**：根据对手行为调整策略的程度

性格维度通过权重系数影响 MCTS 的模拟策略和最终决策评分。

## 五、界面设计（pygame）
- **布局**：椭圆形扑克桌居中，玩家座位环绕桌面排列
- **元素**：
  - 玩家信息卡（姓名、筹码、当前下注、状态）
  - 公共牌区域（Flop / Turn / River）
  - 底池显示
  - 庄家按钮（Dealer Button）、小盲/大盲标记
  - 玩家手牌（人类玩家正面朝上，AI 玩家背面朝上，Showdown 时翻开）
- **操作面板**：人类玩家回合显示可执行动作按钮（Fold / Check / Call / Raise / All-in）+ 滑块选择加注金额
- **动画**：发牌动画、筹码移动动画、翻牌动画（简洁流畅，不过度复杂）
- **音效**：
  - 发牌音效
  - 筹码下注/移动音效
  - 翻牌音效
  - 玩家行动提示音效（轮到人类玩家时）
  - 获胜/淘汰音效
  - 背景音乐（可开关）
- **分辨率**：1280×720，支持窗口缩放
- **配色**：经典绿色赌桌 + 木质边框风格

## 六、玩家设置与存档系统

### 人类玩家
- 游戏开始时可选玩家数量（2-8）
- 人类玩家固定 1 人，其余为 AI
- 人类玩家座位固定在底部中央
- 可选 AI 难度（简单/普通/困难），影响 MCTS 模拟次数和性格极端程度

### 银行系统
- 人类玩家初始拥有 **10,000 筹码** 存在银行中
- 开局前从银行取款进入游戏，游戏结束后筹码存回银行
- 银行余额持久化存储（存档）

### AI 角色池
- 预设 30-50 个 AI 角色，每个角色有独立银行余额（持久化）
- 每局随机选取若干 AI 角色入座，带入筹码从其银行扣除
- 游戏结束后 AI 角色筹码结算回银行
- 可在角色管理界面查看所有 AI 角色的姓名、性格、银行余额、历史战绩

### 存档功能
- 自动存档：每局游戏结束后自动保存（人类银行余额、AI 角色银行余额、统计数据）
- 存档文件存储在本地（JSON 格式）
- 支持手动重置存档（恢复初始状态）

## 七、游戏记录与统计
- **手牌历史**：记录每手牌的完整动作序列，可查看
- **实时统计**：当前 session 的手牌数、人类玩家盈亏、胜率
- **AI 角色战绩**：每个 AI 角色的历史参与局数、胜率、总盈亏
- **日志**：控制台输出关键事件日志（便于调试）

│   ├── deck.py          # 牌组（标准/短牌）

│   └── character_pool.py # AI角色池管理
├── ui/                  # 界面模块
│   ├── renderer.py      # 渲染器
│   ├── animations.py    # 动画
│   ├── audio.py         # 音效管理
│   ├── assets.py        # 资源加载（图片/音效）
│   └── components.py    # UI 组件（按钮、滑块等）
├── data/                # 数据目录
│   ├── save.json        # 存档文件

├── config.py            # 配置（盲注、筹码、分辨率等）

## 九、开发优先级
1. **P0 - 核心引擎**：牌组（标准+短牌）、牌型评估、游戏流程、下注逻辑
2. **P0 - 基础 UI**：桌面渲染、玩家信息、手牌显示、操作面板
3. **P1 - AI（多人模式）**：MCTS + 性格矩阵 + AI角色池
4. **P1 - 动画与音效**：发牌、筹码、翻牌动画 + 全部音效
5. **P2 - AI（2人模式）**：高级AI算法实现（比MCTS更聪明）
6. **P2 - 存档与银行系统**：银行存取、AI角色持久化、自动存档
7. **P2 - 统计与记录**：手牌历史、实时统计、AI角色战绩
8. **P3 - 设置菜单**：盲注选择、模式选择、短牌切换、AI难度选择
```
